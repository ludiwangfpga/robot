在第二章所述的双边遥操作系统架构中，FPGA 承担了从关节角度采样到电机 PWM 输出的完整闭环计算任务。与传统方案将运动学、力控制和电机驱动分别部署于 CPU、DSP 和专用驱动板不同，本研究将上述全部算法统一实现于单片 FPGA 内部，形成一条无中断的硬件计算流水线。本章将沿该流水线的数据流方向，依次介绍各阶段的控制算法原理与数学模型。

如图 3.X 所示，该流水线按功能划分为三部分。第一部分为机械臂控制算法（3.1 节），构成从关节角度到电流指令的主信号链：编码器采集六个关节角度 q ∈ ℝ⁶；正运动学（FK）模块根据 D-H 参数将关节角映射为末端位姿 x ∈ ℝ⁶[1]；解析雅可比模块复用 FK 中间变换矩阵构建雅可比矩阵 J ∈ ℝ⁶ˣ⁶；速度估计模块通过后向差分与低通滤波得到末端速度 ẋ ∈ ℝ⁶；阻抗控制模块按 F = K_s(x_d − x) + D_s(ẋ_d − ẋ) 生成期望交互力 F ∈ ℝ⁶[2]；雅可比转置映射通过 τ = J^T F 将任务空间力转换为关节力矩[3]；力矩-电流转换将力矩除以电机力矩常数 K_t 得到 q 轴电流参考值 i_q^*。整条主链路可概括为：

q → FK → x, J → ẋ → F → τ → i_q^*

各算法的 FPGA 流水线时序与硬件实现细节将在第五章讨论。

3.1 正运动学与解析雅可比

正运动学（Forward Kinematics, FK）是串联机械臂控制的基础，其任务是将关节空间的角度向量 q ∈ ℝ⁶ 映射为末端执行器在任务空间中的位姿 x ∈ ℝ⁶。本节介绍基于 Denavit-Hartenberg（D-H）参数的正运动学算法，以及作为其副产品高效计算解析雅可比矩阵的方法。

3.1.1 D-H 参数与坐标系定义

本研究采用标准 D-H 参数法对 6-DOF EPDobot 机械臂进行运动学建模[1]，D-H 参数根据 EPDobot 机械臂技术手册确定。D-H 参数法通过四个参数——连杆偏距 d_i、连杆长度 a_i、连杆扭角 α_i 和关节角 θ_i——描述相邻关节坐标系之间的齐次变换关系。对于旋转关节，θ_i 为变量，其余三个参数为常数。

表3.1 EPDobot机械臂D-H参数
关节	d_i(m)	a_i(m)	α_i (rad)	θ_i
1	0.166	0.055	π/2	q_1
2	0	0.200	0	q_2
3	0	0.056	π/2	q_3
4	0.192	0	-π/2	q_4
5	0	0	π/2	q_5
6	0.055	0	0	q_6

该机械臂的连杆结构属于典型的 6R 构型：前三个关节构成肩-肘-腕的位置子链，工作空间覆盖范围由最长连杆 a₂ = 0.200 m 决定；后三个关节构成球形手腕（spherical wrist）的姿态子链，其中关节 4-5-6 的旋转轴依次交替正交（α₄ = −π/2, α₅ = π/2），使末端执行器可在任意位置实现完整的姿态调节。

3.1.2 正运动学递推算法

根据标准 D-H 参数法，从第 i−1 个坐标系到第 i 个坐标系的齐次变换矩阵为[1]：

T_{i-1}^i = [cosθ_i, -sinθ_i cosα_i, sinθ_i sinα_i, a_i cosθ_i;
             sinθ_i, cosθ_i cosα_i, -cosθ_i sinα_i, a_i sinθ_i;
             0, sinα_i, cosα_i, d_i;
             0, 0, 0, 1]

末端执行器相对于基座坐标系的位姿通过六个齐次变换矩阵的连乘获得：

T_0^6 = T_0^1 · T_1^2 · T_2^3 · T_3^4 · T_4^5 · T_5^6

该矩阵的第四列前三个元素即为末端执行器的三维位置 p_e = [p_x, p_y, p_z]^T，左上角 3×3 子矩阵 R_0^6 = [r_{ij}] 为姿态旋转矩阵。末端执行器的姿态用 ZYX 欧拉角表示[1]：

r_z = atan2(r_{10}, r_{00})
r_y = atan2(-r_{20}, √(r_{00}² + r_{10}²))
r_x = atan2(r_{21}, r_{22})

计算流程为：依次对每个关节 i = 1, ..., 6，先通过 CORDIC 算法计算 sin θ_i、cos θ_i 和 sin α_i、cos α_i，按式 (3.1) 构建 T_{i-1}^{i}，右乘至累积矩阵 T_{acc}（初始为 I₄）。六个关节处理完毕后从 T_0^6 中提取位置和姿态。

CORDIC（COordinate Rotation Digital Computer）算法通过一系列预定义的旋转角度 αₖ = arctan(2⁻ᵏ) 逐步逼近目标角度[2]。对于 sincos 模式（旋转模式），迭代公式为：

x_{k+1} = x_k - σ_k · 2^{-k} · y_k
y_{k+1} = y_k + σ_k · 2^{-k} · x_k
z_{k+1} = z_k - σ_k · arctan(2^{-k})

其中 σ_k = sign(z_k)。初始值 x₀ = 1/K_n（增益补偿因子），y₀ = 0，z₀ = θ，经 n 次迭代后 x_n ≈ cos θ，y_n ≈ sin θ。向量模式（atan2）中 σ_k = −sign(y_k)，初始值 x₀, y₀ 为输入坐标，z₀ = 0，迭代完成后 z_n ≈ atan2(y₀, x₀)，用于式 (3.3) 的欧拉角提取。全部运算仅需移位和加减法，不依赖硬件乘法器。本实现采用 16 次迭代，精度达 16 位有效位，与 Q16.16 定点格式匹配。

正运动学模块总延迟为 382 时钟周期（约 1.51 μs @ 253 MHz）。位置计算精度通过 1004 个蒙特卡洛采样测试用例验证，平均线性误差仅 0.0188 mm，远优于第二章设定的 0.15 mm 精度要求。

3.1.3 解析雅可比

雅可比矩阵 J ∈ ℝ⁶ˣ⁶ 建立了关节速度 q̇ 与末端速度 ẋ 之间的线性映射关系[3]：

ẋ = J(q) · q̇ (3.5)

对于全旋转关节的串联机械臂，雅可比矩阵第 i 列的几何意义明确[1]：

J_i = [(z_{i-1} × (p_e - p_{i-1})); z_{i-1}] (3.6)

其中 z_{i-1} 和 p_{i-1} 分别为第 i−1 个坐标系的 z 轴方向和原点位置。上半部分对应该关节引起的末端线速度贡献，下半部分对应角速度贡献。

本设计利用正运动学递推过程中已计算的各坐标系原点和 z 轴向量，直接按式 (3.6) 组装雅可比矩阵，无需额外的坐标变换计算。该模块延迟 6 周期，与 FK 并行执行。

3.1.4 速度估计

末端速度 ẋ 通过后向差分加一阶 IIR 低通滤波获得[5]：

(ẋ_{filt})[n] = α · (x[n] - x[n-1])/Δt + (1-α) · (ẋ_{filt})[n-1] (3.7)

滤波系数 α = 0.3，六自由度并行处理，延迟 12 周期。

3.1.5 任务空间阻抗控制

阻抗控制在任务空间中定义弹簧-阻尼器模型[6][3]。为简化 FPGA 实现，本研究采用纯弹簧-阻尼模型，忽略惯性项 M_s，该简化在低速操作场景下成立：

F = K_s · (x_d - x) + D_s · (ẋ_d - ẋ) (3.8)

K_s 和 D_s 为对角矩阵，六自由度独立可调，输出 F ∈ ℝ⁶ 为期望交互力/力矩。基础模式下为固定值，3.2 节将介绍其在线自适应调节。延迟 3 周期。

3.1.6 雅可比转置映射与力矩-电流转换

任务空间力 F 通过雅可比转置映射为关节力矩[7]：

τ = J^T · F (3.9)

矩阵转置为纯线路交换（零延迟），矩阵-向量乘法共 8 周期。关节力矩经力矩常数倒数转换为 q 轴电流参考值：

i_q^* = τ · K_t^{-1} (3.10)

六通道并行，延迟 2 周期。至此完成从 q 到 i_q^* 的完整链路，主链路总延迟约 413 周期（约 1.63 μs）。

3.2 自适应阻抗控制

在3.1.5节的阻抗控制律采用固定参数 K_s 和 D_s，但实际遥操作中环境刚度 K_env 未知且时变[1]。固定参数存在根本矛盾：高刚度在硬接触时产生过大接触力，低刚度在软环境中导致跟踪误差增大。自适应阻抗控制通过在线估计 K_env 并动态调整参数来解决这一矛盾，理论分析表明可将接触力降低约 75%，具体实验验证见第五章。

3.2.1 环境接触力建模

环境接触力在关节空间中建模为[2]：

τ_{ext,i} = K_{env,i} · e_{q,i} (i=1,2,...,6) (3.11)

其中 τ_{ext,i} 为外部扰动力矩，e_{q,i} = q_{d,i} − q_i 为关节位置误差。对角化假设使六个关节完全解耦，每个关节独立运行一个标量 RLS 估计器。

3.2.2 无传感器力矩获取

外部力矩 τ_{ext} 通过 FOC 电流环反馈的实际 q 轴电流反推获得[3]：

τ_{meas,i} = K_{t,i} · i_{q,actual,i} (3.12)
τ_{ext,i} = τ_{meas,i} - τ_{cmd,i} (3.13)

其中 τ_{cmd,i} 为阻抗控制器输出的力矩指令。该方法完全复用 FOC 已有的电流传感器，无需额外力/力矩传感器，力估计精度达 0.05 N，满足第二章力精度要求。

3.2.3 RLS 在线估计

每个关节的 K_{env,i} 通过带遗忘因子的标量 RLS 算法在线估计[4]：

γ_i = (P_i · e_{q,i}) / (λ + e_{q,i} · P_i · e_{q,i}) (3.14)
K_{env,i} ← K_{env,i} + γ_i · (τ_{ext,i} - K_{env,i} · e_{q,i}) (3.15)
P_i ← 1/λ · (P_i - γ_i · e_{q,i} · P_i) (3.16)

遗忘因子 λ = 0.98，对应等效数据窗口 50 次采样（约 200 μs），可在 50 次迭代内收敛至真实值的 95%。

3.2.4 死区保护与自适应规则

自由空间运动时 e_{q,i} 极小，RLS 增益发散。当 |e_{q,i}| < ε_dz = 0.001 rad 时，冻结 RLS 更新，输出名义参数。

接触状态下，刚度按反比例规则调整[5]：

K_{s,out,i} = clamp(K_{s,nom,i} × (K_{s,nom,i} / K_{env,i}), K_{s,min}, K_{s,max}) (3.17)

环境越硬则控制器越软，环境越软则控制器越硬，维持闭环总体刚度近似恒定。该规则使控制器刚度与环境刚度成反比，维持两者乘积近似恒定。理论分析表明该自适应策略可将硬接触时的接触力降低约 75%，具体实验验证见第五章。阻尼同步调整以保持阻尼比:

D_{s,out,i} = D_{s,nom,i} · K_{s,out,i} / K_{s,nom,i} (3.18)

自适应模块总延迟 48 周期（约 0.19 μs），与 FK（382 周期）并行执行，不增加总流水线延迟。

3.3 FOC 电机驱动

阻抗控制链输出的电流参考值 i_q^* 通过磁场定向控制（FOC）转换为驱动电机的 PWM 信号。本节介绍 FOC 各环节的数学原理、六通道并行架构及跨时钟域传递方案。

3.3.1 FOC 控制环路

FOC 通过坐标变换将三相交流电流分解为磁通分量 i_d 和力矩分量 i_q，实现独立控制[1]。i_d = 0 时电磁力矩仅由 i_q 决定，力矩/电流比最大。

(i_a, i_b, i_c) —Clark→ (i_α, i_β) —Park→ (i_d, i_q) —PI→ (V_d, V_q) —inv-Park→ (V_α, V_β) —SVPWM→ PWM

其中 Clark 变换将三相电流投影至两相静止坐标系，Park 变换以转子电角度 ψ = p · θ_m 将其旋转至与转子磁场同步的 d-q 坐标系，PI 控制器分别对 d、q 轴电流误差进行闭环调节（积分项带抗饱和保护），逆 Park 变换将输出电压变换回静止坐标系后，由 SVPWM 以 7 段式调制合成三相电压输出。

3.3.2 六通道并行架构

六关节电机并行执行 FOC 运算，每通道独立维护 d-q 轴 PI 控制器状态。电流采样以 36 kHz 触发，六通道时分复用同一组坐标变换硬件，状态寄存器按通道隔离。

3.3.3 跨时钟域传递

阻抗控制运行于 253 MHz，FOC 运行于 36 kHz（由 PWM 周期决定）。i_q^* 通过异步 FIFO 从高频域传递至低频域，FIFO 深度为 4，确保数据不丢失。FIFO 写使能为阻抗控制完成信号，读使能为 FOC 采样触发信号。

3.4 本章小结

本章完整阐述了 FPGA 硬件计算流水线中的控制算法原理：

(1) 正运动学采用 D-H 参数递推与 CORDIC 算法，位置计算精度达 0.0188 mm，远优于 0.15 mm 的系统要求；

(2) 解析雅可比复用 FK 中间结果高效计算，速度估计采用后向差分加低通滤波；

(3) 任务空间阻抗控制定义弹簧-阻尼器模型，通过雅可比转置映射为关节力矩；

(4) 自适应阻抗控制通过 RLS 在线估计环境刚度，动态调整阻抗参数，理论分析表明可将接触力降低约 75%，力估计精度达 0.05 N；

(5) FOC 电机驱动完成电流到 PWM 的转换，六通道并行执行。

整个控制链路中，主链路延迟约 1.63 μs，FOC 驱动延迟约 5.5 μs，系统总延迟约 7.2 μs，满足第二章 < 10 μs 的延迟要求。各算法的 FPGA 硬件实现细节将在第五章详细讨论。
