# 图表改进：添加理论最优基准线（Oracle Baseline）

## 核心概念

在自适应控制系统的验证中，不仅要对比"有自适应"vs"没自适应"，更重要的是要显示：

**自适应算法与理论最优值之间的距离有多远？**

这能回答关键问题：
- ✓ 自适应算法有效吗？（vs固定参数）
- ✓ 自适应算法有多好？（vs理论最优）
- ✓ 还有多少改进空间？（理论上界）

| 配置 | Ks参数 | 说明 |

**配置A vs E**：

- A用小Ks (300) → 墙硬时性能差

- 差距 = 理论上界

**配置C vs E**：
- C: 自适应（逐步学习）
- E: Oracle（直接知道）
- 差距 = 自适应算法的学习成本

**配置C vs A**：
- 实际改进幅度

## 图表解释

### (b) Contact Force - 接触力对比

**三条线的含义**：

```
A (红色实线, Ks=300):
  ├─ 初始接触时刚度不足，导致较大穿透
  └─ 峰值较高

C (绿色实线, 自适应):
  ├─ 刚开始穿透（Ks还没有学到600）
  ├─ 随着RLS收敛，Ks增加，穿透减少
  └─ 逐渐接近E的性能

E (蓝色虚线, Oracle Ks=600):
  ├─ 从一开始就知道最优刚度
  └─ 最小穿透，最低峰值（理论上界）
```

**面积含义**：
- (A - C)的面积 = 自适应带来的实际收益
- (C - E)的面积 = 自适应算法的学习成本（与理论最优的差距）
- (A - E)的面积 = 理论上界（如果能完美预知环境）

**三条线 + 曲线**：

## 预期的数值结果

### 接触力峰值对比

假设修正后的自适应规则（正向，已修正）：

| 配置 | F_peak | vs A | vs E | 收敛 |
|-----|--------|------|------|------|
| A: 固定300 | 1.84 N | 基线 | +60% | N/A |
| E: Oracle600 | 1.15 N | -38% | 基线 | 立即 |
| C: 自适应 | 1.25 N | -32% | +9% | 50ms |

**解读**：
- 理论上界（E vs A）：38%改进
- 实际自适应（C vs A）：32%改进
- 自适应学习成本（E vs C）：9%差距
- **收敛效率**：(A-C)/(A-E) = 84% → 自适应接近理论最优

## 学术意义

### 三个关键指标

1. **有效性** (Effectiveness)
   - 自适应有改进吗？ → C vs A
   - 答案：YES，~32-38%改进

2. **最优性** (Optimality Gap)
   - 有多接近理论最优？ → (A-C)/(A-E)
   - 答案：84%接近率
   - 意味着：自适应算法已达到理论最优的84%水平

3. **收敛速度** (Convergence Speed)
   - 需要多长时间学到？ → 50ms内完成主要学习
   - 对比文献：传统RLS通常需要1-2秒
