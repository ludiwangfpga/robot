第二章 双边遥操作系统架构

2.1 系统总体架构

双边遥操作系统采用主从架构，由操作端（主手）和执行端（从手）组成。主手接收操作者的运动指令，通过通信链路传输至从手；从手在远端环境中执行任务，同时将感知到的环境力反馈回主手，使操作者获得力觉临场感。

本系统的核心特点包括：
- 单FPGA实现完整控制链：从编码器输入到PWM输出的全部算法在单片FPGA内完成
- 高频实时控制：253 kHz控制频率，3.95 μs端到端延迟
- 自适应阻抗控制：根据环境刚度自动调整控制参数
- ROS2分层架构：FPGA负责实时控制，ROS2负责高层规划

2.2 双边遥操作的精度要求

2.2.1 位置精度要求

双边遥操作对位置精度有严格要求，以确保操作者能够精确控制远端执行器：

- 运动学计算误差 < 0.15 mm
  - 正运动学计算精度直接影响末端执行器定位
  - 采用Q16.16定点格式，理论分辨率约0.015 mm

- 闭环跟踪精度：
  - 软环境（K_env < 500 N/m）：< 1 mm
  - 硬环境（K_env > 2000 N/m）：< 0.1 mm
  - 硬环境需要更高精度以避免过度穿透

2.2.2 力控制精度要求

力反馈的精度决定了操作者的力觉临场感质量：

- 力估计误差 < 10%
  - 基于电机电流反推的力矩估计精度
  - 受电机参数（Kt）精度和温度漂移影响

- 力分辨率 < 0.05 N
  - 人类手指的力觉JND（最小可觉差）约为0.5 N
  - 系统分辨率应优于JND一个数量级

- 力带宽 > 100 Hz
  - 人类力觉感知带宽约为0-100 Hz
  - 系统力控制带宽需覆盖此范围

2.2.3 透明度与精度权衡

固定阻抗参数难以同时满足软硬环境的精度要求：

- 高刚度参数（Ks大）：
  - 优点：硬环境下跟踪精度高
  - 缺点：软环境下接触力大，安全性差

- 低刚度参数（Ks小）：
  - 优点：软环境下柔顺性好，安全性高
  - 缺点：硬环境下跟踪精度差

这一矛盾需要通过自适应阻抗控制解决，根据环境刚度实时调整控制参数。

2.3 从手侧控制链路数据流

从手侧控制链路的数据流如下：

q → FK → x → F → τ → i_q → FOC → PWM

各阶段说明：
1. q：编码器采集的六个关节角度
2. FK：正运动学计算，将关节角映射为末端位姿
3. x：末端执行器在任务空间中的位置和姿态
4. F：阻抗控制输出的任务空间力/力矩
5. τ：雅可比转置映射得到的关节空间力矩
6. i_q：力矩-电流转换后的q轴电流指令
7. FOC：磁场定向控制，输出三相PWM信号
8. PWM：驱动电机逆变器的脉宽调制信号

该链路在FPGA内部以流水线方式执行，全部计算在3.95 μs内完成。

2.4 自适应阻抗控制的精度需求分析

自适应阻抗控制需要满足以下精度要求：

- 环境刚度估计误差 < 30%
  - RLS算法收敛后的估计精度
  - 影响自适应控制的有效性

- 收敛速度 < 50 ms
  - 从接触开始到参数稳定的响应时间
  - 需快于环境变化速度

- 参数调节粒度 < 5%
  - 阻抗参数Ks的调整步长
  - 避免参数振荡

2.5 FPGA + ROS2 分层架构

本系统采用分层架构设计：

**FPGA层（实时控制层）**：
- 控制频率：253 kHz
- 延迟：3.95 μs
- 功能：运动学计算、阻抗控制、电机驱动
- 接口：AXI-Lite寄存器（参数配置）、编码器输入、PWM输出

**ROS2层（规划决策层）**：
- 控制频率：~10 Hz
- 功能：路径规划、任务调度、人机交互
- 通信：DDS中间件，支持多设备分布式部署

**分层优势**：
- 实时性与灵活性兼顾
- 硬件加速与软件可扩展性结合
- 通过AXI-Lite接口实现参数实时配置

2.6 本章小结

本章介绍了双边遥操作系统的总体架构和精度要求：

1. 系统采用主从双边架构，实现力觉临场感
2. 位置精度要求：运动学误差<0.15 mm，闭环跟踪软环境<1 mm、硬环境<0.1 mm
3. 力控制精度要求：估计误差<10%，分辨率<0.05 N，带宽>100 Hz
4. 控制链路数据流：q → FK → x → F → τ → i_q → FOC → PWM
5. 自适应控制精度：刚度估计误差<30%，收敛<50 ms，调节粒度<5%
6. FPGA+ROS2分层架构实现实时控制与高层规划的分离
